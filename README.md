## 課題提出用情報

### 制作アプリのタイトル
Atora（アトラ） | Path of Sages

### 制作アプリの説明（40文字程度）
自然と調和し自己をチューニングする、貴方だけの旅のキュレーター・アーカイブ
- Concept / コンセプト
Atoraは、単なる観光ガイドではなく、「貴方だけの旅のキュレーター」として機能します。
現代人が抱える日常の悩み（自己・他者・生業）を、日本の歴史の中で自らを研鑽し、自然と深く対話してきた賢人・偉人たちの視点とマッチングさせます。彼らがその土地で何を感じ、どのように自然と調和したのか。そのプロセスを辿ることで、ユーザー自身の波長を整え（チューニング）、新たな気づきを得るための精神的な旅路をキュレーションします。

### 工夫した点・こだわった点
- **デザインの静寂感**: 賢人の思考に触れる体験を壊さないよう、MontserratとNoto Serif JPを組み合わせた繊細なタイポグラフィと、モノトーン基調の Leaflet.js マップデザインを徹底しました。
- **パーソナライズ機能**: ユーザーが入力した悩みの系統をPHPで解析し、その時に最適な賢人ルートが地図上のリスト上部に優先表示されるようロジックを組みました。
- **没入の設計**: 単なる観光地紹介にならないよう、DBに「推奨ルート（行程）」を持たせ、地図のクリックから滑らかなアニメーションで具体的な行動指針を提示するUI/UXにこだわりました。

### 次回トライしたこと（または機能）
- **データの厚み**: 賢人の一言（Quote）や、現地で行う具体的な「3ステップの儀式（Ritual）」のカラムを追加し、コンテンツの解像度を高めたいです。
- **管理者画面の拡充**: 緯度経度や物語をより直感的に登録・編集できる専用のダッシュボード機能の構築。
- **感情分析**: ユーザーの悩みのテキストから感情を分析し、より精度の高いレコメンドエンジンを実装したいです。

### 備考（感想、シェアしたいこと等なんでも）
PHPとデータベースを連携させることで、ユーザーの「今」の気持ちに合わせた動的な体験作りができる楽しさを実感しました。特に、地図上のピンをクリックした際のモーダルの連動と、そこに至るまでの「余白」を意識したCSSの調整に苦労しましたが、その分「Atora」らしい静かな空気感を作ることができたと感じています。まだまだ目指す形には遠いですが、地道に進めます。

普段、Expedia・agoda・じゃらん等々の旅行系アプリを出張で使っているものの、プライベートでは情報が多く見たくない・・・思考力、脳のエネルギーを極力使わず、仕事モードの脳にならず、知的好奇心などは元気の源のため生きていてほしい。そんな諸々の願いを詰め込んだセカイを創りたいところです。
